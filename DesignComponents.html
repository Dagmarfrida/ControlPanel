Components

Components are the reusable buildings blocks through out the WeUse material. We aim to be intuitive in our work and for positive user experience Every components that is boxed should have 8px corner radius, 15px if it is a big component.

Buttons

Buttons should have minimum width of 80px. Where multiple buttons appear, all of them should be the same size. If possible dependent on text length the horizontal paddings should be minimum 12px. Vertical padding should be minimum 10px. Text should have first letter of every word capitalised.

Primary Button

Should be used sparingly and only when there is need to cancel or delete something

Primary Cancel Button

Should be used sparingly and only when there is need to cancel or delete something

Secondary button

Can be used f.eks. to navigate to other content on the same page

Tertiary button

Should be support to Primary or Secondary buttons or for actions where Primary or Secondary buttons are not needed They er not the main focus but should be easy to discover.

Button with Icons

Icon always displays after the text and the icon should be similar to cap height of the font. Text should be left aligned and icon right aligned with the sem padding

Checkbox

Checkboxes are used when the user can choose between multiple choices where he/she can select one or more.

Tableview

Tableview is used everywhere lists are displayed. It follows grid of 145 px columns with 16 px gutter. Where text is centered.

Input fields

Input field is an input that users can fill out a text or edit it. Corner radius is 15px. Border color is Whale 40

Default input field

Input field with message

Message is a way to let the user know if something worked or not. It has different color depending on the message. The fonts size of message is 14px

Pop ups

Pop Up messages appear when user needs to confirm his action or when he is deleting something that needs to be confirmed. They have different color depending on their action.
